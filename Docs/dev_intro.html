Developer's Guide

Introduction

This software is free and open-source. It is designed to grow over time to accept new formats, run new tasks and have new purposes. As such, this guide should provide the necessary information to help devopers understand the software and expand it.

How to add Geometry Interfaces

Geometry Interfaces (GIs) are described in Settings/Flow.xml and linked by their GeometryInterfaceID (GIID) in Constants/Constants.cs. Adding a GI requires adding a geometryInterface tag to Flow.xml and making the following changes:

  1. Add a new item to the GeometryInterfaceID enum in Constants.cs
  2. Add a unique string to map to this item in the StringToGeometryInterfaceID dictionary
  3. Add a "name" attribute to geometryInterface - this must correspond with the string provided in the StringToGeometryInterfaceID dictionary
  4. Give the GI a short name to appear in the GUI with title
  5. Give the GI a full name to appear in the popup with fullName
  6. Give the GI a description depending on what error state it is with disabledDescription, completedDescription, okDescription, loadingDescription, warningDescription, and errorDescription
  7. Set the coordinates of new GI with position
  8. Add Residue Checkers with residueCheck. Each Residue Checker will need a name that corresponds to the StringToResidueCheckerID dictionary in Constants.cs, and the Geometry Interface Status if the check fails
  9. Add Atom Checkers with atomCheck. Each Atom Checker will need a name that corresponds to the StringToAtomCheckerID dictionary in Constants.cs, and the Geometry Interface Status if the check fails
