Introduction

Overview

What is ONIOM-Flow?

ONIOM-Flow is a Graphical User Interface designed to facilitate ONIOM calculations for biological systems. It is intended to take Protein Database (PDB) files with Non-Standard Residues and produce Gaussian Job Files (.gjf/.com) with the necessary partial charges and parameters.

What isn't ONIOM-Flow?

ONIOM-Flow is not black-box software. While it is designed to remove some of the processes that are normally needed for setting up ONIOM calculations, it still requires user input at various stages.

Structure

The interface consists of two classes: Geometry Interfaces (GIs) and Arrows.

  • GIs provide a way to access geometries, called Atoms in the case of this software. They contain the current state of the Atoms, and allow loading and saving through a variety of formats including the in-house XAT format. They also provide methods of validation to allow the user to ensure the Atoms are in the correct state to continue.
  • Arrows link GIs. Clicking on an active Arrow (green or blue in colour) will open a popup listing the Tasks that will be performed.

History

This software was borne out of a Python project started by Clyde Fare in 2014 as part of his gausspy project, and continued by Tristan Mackenzie. This project attempted to link together several pieces of software written by others to produce a near-black box piece of code that could protonate and parameterise proteins for ONIOM calculations. However, this was abandoned due to impracticalities created by the incompatibilities between the software (some had strict definitions for file formats/PDB IDs etc) which lead to the project expanding to the point where Python was no longer a feasible language. The need for graphical feedback and cross-platform compatibility to the user resulted in Unity being the platform of choice for development.
